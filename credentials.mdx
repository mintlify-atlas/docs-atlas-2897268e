---
title: "Credentials"
description: "Configure AzuraCast API credentials in n8n."
---

The **AzuraCast** node uses the **AzuraCast API** credential type (`renebelloAzuraCastApi`). You create this credential once and reuse it across all AzuraCast nodes in your workspace.

## Credential fields

<ParamField path="body.baseUrl" type="string" required placeholder="https://radio.example.com">
  The root URL of your AzuraCast installation. Do not include a trailing slash or the `/api` path — the node appends `/api` automatically.

  Example: `https://radio.example.com`
</ParamField>

<ParamField path="body.apiKey" type="string">
  Your AzuraCast API key. This field is optional for public endpoints but required for any protected station or administration operation.

  Generate a key in AzuraCast under **My Account** → **API Keys**. The value is stored encrypted in n8n.
</ParamField>

## Base URL normalization

The node normalizes the base URL before making requests:

1. Any trailing slashes are removed.
2. If the URL does not already end with `/api`, the path `/api` is appended.

For example, `https://radio.example.com/` becomes `https://radio.example.com/api`, and `https://radio.example.com/api/` stays as `https://radio.example.com/api`.

You do not need to include `/api` in the Base URL field.

## Authentication headers

When an API key is set, every request includes two headers:

```http
X-API-Key: <your-api-key>
Authorization: Bearer <your-api-key>
```

AzuraCast accepts either header format; both are sent for maximum compatibility. When no API key is configured, no authentication headers are added and only public endpoints will respond successfully.

## Credential test

When you click **Test credential** in n8n, the node runs a verification request:

| Condition | Endpoint tested |
|---|---|
| No API key set | `GET /api/nowplaying` (public endpoint) |
| API key set | `GET /api/frontend/account/me` (authenticated endpoint) |

A successful test confirms both the base URL is reachable and (when a key is provided) the key is valid.

<Tip>
  If you only need to access public data — such as now-playing status or station listings — you can create a credential with just the Base URL and leave API Key blank. Public endpoints work without authentication.
</Tip>

<Warning>
  Make sure the Base URL points to the root of your AzuraCast domain, not to the admin panel or a specific page. The node constructs all API paths from this root.
</Warning>
