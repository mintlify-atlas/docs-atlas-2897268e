This guide walks you through fetching live now-playing data from your AzuraCast station — no API key required.

<Steps>
  <Step title="Install the node">
    Follow the [Installation guide](/installation) to add `@renebello/n8n-nodes-azuracast` to your n8n instance.
  </Step>

  <Step title="Add the AzuraCast node to your canvas">
    Open a workflow in n8n. Click the **+** button to add a node, then search for **AzuraCast** and select it.
  </Step>

  <Step title="Create AzuraCast API credentials">
    When prompted for credentials, click **Create new credential**.

    - **Base URL**: enter the root URL of your AzuraCast installation, e.g. `https://radio.example.com`
    - **API Key**: leave blank for public endpoints, or enter your key for protected ones

    Click **Save** to store the credential. See the [Credentials guide](/credentials) for details.
  </Step>

  <Step title="Select a Resource">
    In the node parameters, open the **Resource** dropdown and choose **Public Now Playing**.

    Resources map to the official AzuraCast API tag groups. Other examples include `Stations Media`, `Stations Playlists`, and `Administration Users`.
  </Step>

  <Step title="Select an Action">
    Open the **Action** dropdown and choose **Get All Now Playing**.

    This calls `GET /api/nowplaying` and returns the current live status for all stations, including track info, listener counts, and stream URLs. It requires no path or body parameters.
  </Step>

  <Step title="Execute the node">
    Click **Test step** (or press `Ctrl+Enter`). The node will query your AzuraCast instance and return the now-playing data as JSON items in the output panel.
  </Step>
</Steps>

## Example output

A successful `Get All Now Playing` response returns an array of station objects. Each item includes:

```json
{
  "station": {
    "id": 1,
    "name": "My Radio Station",
    "shortcode": "my_radio"
  },
  "now_playing": {
    "song": {
      "title": "Example Track",
      "artist": "Example Artist"
    },
    "duration": 214,
    "elapsed": 43
  },
  "listeners": {
    "current": 12
  }
}
```

## Public vs protected endpoints

<Note>
  The **Public Now Playing** and **Public Stations** resources do not require an API key. All other resource groups (administration, station management, reporting) require a valid API key set in your credentials.
</Note>

To access protected operations, add your AzuraCast API key to the credential. The node will automatically include the required authentication headers on every request.
